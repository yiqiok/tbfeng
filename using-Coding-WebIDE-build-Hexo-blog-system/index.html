使用Coding WebIDE搭建云端Hexo博客系统

2017-04-18 | tech

提起Hexo博客系统，相信很多使用过静态博客生成器的人都不会陌生。我最初是使用Hexo在GitHub Pages上搭建自己的静态博客，非常方便，折腾了不少主题，在优化上花了很多时间，写了几篇文章。不过后来换了电脑，也就懒得再折腾一次。只是最近心里又萌生了写点东西的冲动，本来也可以开个微信公众号随便写写，但心底里还是觉得微信公众号太封闭了，还是写博客有情怀。这里插一句，我真是挺佩服哪些每天都更新微信公众号文章的作者，每天都坚持写，还都不重样，不管什么新闻，热点都能就着写出自己的观点来。相比之下，我没有那么多观点要向大家展示，因为我觉得有些事情压根和我没关系，或许是我内心老了？不管怎样，我希望永远有渴望，stay young, stay simple, stay naive :)

相比在本地搭建，使用Coding WebIDE搭建Hexo博客系统的优势是你换了电脑后依然可以轻松写博客。我也想过用U盘做一个操作系统，然后安装Hexo,或者直接使用云服务器，那样做一是麻烦，二是成本高。

使用Coding WebIDE搭建Hexo博客系统，需要Coding.net的账号，可以到官网注册。Coding.net和GitHub类似，但它在国内的访问速度更快，而且提供WebIDE。WebIDE官方的简介是：

Coding WebIDE 是 Coding 自主研发的在线集成开发环境 (IDE)。用户可以通过 WebIDE 创建项目的工作空间, 进行在线开发, 调试等操作。同时 WebIDE 集成了 Git 代码版本控制, 用户可以选择 Coding、GitHub、BitBucket、Git@OSC 等任意的代码仓库。 WebIDE 还提供了分享开发环境的功能, 用户可以保存当前的开发环境, 分享给团队的其他成员。

如果你有基本的Linux命令行使用经验更好，此外还需要基础的git知识。当然，上面的这些都不会也没有关系，只要你有点耐心，并会用搜索引擎来寻找遇到的问题的解决方法就行，用好Google吧。

我不是专业搞IT的，做这些都不难，相信你也能学会。

建立仓库，搭建IDE环境

在Coding.net注册了账号后，我们可以点击‘+创建新项目’来创建一个仓库,项目名称和描述可以自定义，然后将仓库设为私有，并勾选‘启用README.md文件初始化项目’，最后点击绿色的’创建项目‘按钮。

仓库创建完成之后，点击“IDE”，

然后单击“+ 新建工作空间”:

会出现这个页面：

点击“同步仓库”，同步仓库完成后。你就能看到刚才建立的仓库了。
然后点击同步过来的仓库，你就进入到配置workspace的页面啦。

这里我们都选择最低档配置就可以了，对这个项目够用的，放心。

建立和使用workspace是需要码币的，刚开使会送些给你，完善个人信息，下载使用App等也可以赚到码币，对于我们写博客来说，码币应该是够用的。

点击下方的‘创建’，等待workspace创建好后，我们就可以单击打开运行它了


然后点击上图右上部‘运行环境’，选择使用ide-tty-nodejs作为运行环境。

到这里IDE环境搭建好了。下一步，我们要安装Hexo。
